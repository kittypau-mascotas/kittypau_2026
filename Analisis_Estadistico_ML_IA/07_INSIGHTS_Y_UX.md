# Insights y UX

## Principio
Todo insight debe responder:
1) qué pasó
2) cuándo (period_start/period_end)
3) por qué (contexto con métricas)
4) qué debería hacer el usuario (acción sugerida)

## Severidad sugerida
- `info`: variación esperable / aviso
- `warning`: desviación relevante / requiere atención
- `alert`: riesgo alto / acción inmediata
